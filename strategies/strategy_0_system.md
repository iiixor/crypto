# Система четырёх стратегий — как всё работает вместе

> Применимо к ситуации: $200 стартовый капитал, преподаватель + Go-разработчик, Bybit/OKX/Binance.
> Документ описывает не отдельные стратегии, а то, как они дополняют друг друга в единой системе.

---

## Ключевая идея

Четыре стратегии не конкурируют — они закрывают разные временны́е горизонты и разные типы рыночных событий.

| Стратегия | Горизонт | Плечо | Событие |
|---|---|---|---|
| **Token Splash Short** | 1–24 часа | 4x | Любой новый листинг с пампом |
| **Launchpool Harvest** | 24–48 часов | 3x | Объявление Launchpool/HODLer (за 4–7 дней) |
| **TGE Short** | 1–7 дней | 2x | Airdrop TGE с >50K получателей |
| **VC-гравитация** | 2–8 недель | 2x | Новый листинг с FDV Ratio > 8x |

Из-за разных горизонтов позиции практически не пересекаются. Token Splash закрывается за 24 часа, Launchpool — за 48, VC-гравитация живёт неделями в фоне. Ты можешь держать **2–3 позиции одновременно** без превышения разумного уровня риска.

---

## Почему именно эта комбинация

**Token Splash** — основной генератор дохода. Высокое плечо, высокая частота, максимальная прибыль за сделку. Требует бота с WebSocket.

**Launchpool Harvest** — надёжный фундамент. Win rate 91.7%, событие известно за 4–7 дней — бот делает всё по расписанию. Идеально для рабочих будней в институте: объявление вышло вечером, бот откроет позицию завтра утром сам.

**VC-гравитация** — пассивный фон. Ты открываешь одну позицию на 4–8 недель и забываешь. Одна проверка в день. Не требует мониторинга в реальном времени. Совместимо с преподаванием и любыми другими делами.

**TGE Short** — ситуативный добавочный доход. Не основная стратегия, но когда событие квалифицируется по всем 5 фильтрам — добавляет в систему 30–80% маржи, которые иначе бы не появились.

## Совместимость стратегий во времени

Типичная неделя при активной работе системы:

```
ПОНЕДЕЛЬНИК
  → Проверяешь анонсы Binance/Bybit (5 минут, бот делает автоматически)
  → Если есть Launchpool анонс → бот планирует открытие шорта через 12–24ч
  → Если в фоне есть VC-гравитация позиция → смотришь цену 1 раз (2 минуты)

ВТОРНИК
  → Launchpool листинг → бот открыл шорт утром сам
  → Ты читаешь лекцию, бот работает
  → Через 24–48ч бот закрылся по TP

СРЕДА
  → Новый листинг на Bybit (Token Splash сигнал)
  → Бот детектирует: RSI > 85 + Volume Spike → открывает шорт
  → Через 2–8 часов закрылся по TP1/TP2

ЧЕТВЕРГ
  → Анонс TGE крупного проекта → проверяешь 5 фильтров
  → Если проходит → бот открывает шорт за 2 часа до TGE

ПЯТНИЦА
  → Подводишь итоги недели: P&L, логи сделок
  → Открываешь VC-гравитация шорт на токен с FDV Ratio > 8x, если появился

ВЫХОДНЫЕ
  → Авторалли, велотрек → бот работает без тебя
```

## Одновременная работа позиций — не страшно

> ⚠️ **Применимо начиная с Фазы 2 ($500+).** В Фазе 1 ($200–$500) работает только ОДНА активная позиция единовременно — это базовое правило защиты капитала. Расчёт ниже показывает ситуацию Фазы 2, когда разрешены две параллельные позиции.

При $500+ капитале (Фаза 2):

**Одновременно открыты:**
- VC-гравитация: 20% капитала × 2x = $100 маржа, $200 нотионал (пример при $500)
- Token Splash: 22.5% капитала × 4x = $112.5 маржа, $450 нотионал

**Суммарное маржинальное обеспечение:** 20% + 22.5% = **42.5% капитала** — это нормально. Биржи дают margin call обычно при >80%.

**Худший сценарий (оба стопа одновременно):**
- VC-гравитация: −10% стоп × 2x × 20% = −4% капитала
- Token Splash: −15% стоп × 4x × 22.5% = −13.5% капитала
- Итого: **−17.5% капитала за один день** — это терпимо, не катастрофа.

## Фазы роста и распределение капитала

### Фаза 1: $200–$500 (первый месяц)

**Принцип:** одна активная позиция в любой момент времени.

Приоритет стратегий:
1. Launchpool Harvest — запускаешь первым (самый простой бот, высокий win rate)
2. Token Splash — подключаешь, когда нет активного Launchpool трейда
3. VC-гравитация и TGE — пропускаешь, капитал нужен для основных двух

Распределение на $200:
```
Каждая сделка = 20–22.5% капитала
Свободный остаток = 77.5–80%
→ Защита от внезапных маржин-коллов и проскальзывания
```

Реалистичный результат: **$200 → $380–450** за первый месяц при 6–8 сделках.

---

### Фаза 2: $500–$1,000 (второй месяц)

**Принцип:** две параллельные позиции.

Распределение:
- Token Splash: 22.5% капитала → основная
- Launchpool Harvest: 20% капитала → фоновая
- Свободный буфер: ~57.5%

Теперь VC-гравитация входит как третья, **параллельная** позиция:
- 15% капитала на долгосрочный шорт с FDV Ratio > 8x
- Это занимает ещё 30$ от $500 = терпимо

Реалистичный результат: **$500 → $900–1,100** за второй месяц.

---

### Фаза 3: $1,000+ (третий месяц и далее)

**Принцип:** все четыре стратегии активны, с чёткими лимитами.

Максимальная одновременная маржа = **60% от капитала** (распределение ниже):

| Стратегия | Маржа от капитала | Нотионал (плечо) |
|---|---|---|
| Token Splash | 22.5% | × 4x |
| Launchpool Harvest | 15% | × 3x |
| TGE Short (если событие) | 12.5% | × 2x |
| VC-гравитация (фон) | 10% | × 2x |
| **Итого** | **60%** | — |
| Свободный буфер | 40% | — |

Худший сценарий (все 4 стопа одновременно):
- Token Splash: -60% × 22.5% = -13.5%
- Launchpool: -36% × 15% = -5.4%
- TGE: -24% × 12.5% = -3%
- VC-gravity: -20% × 10% = -2%
- **Итого потеря капитала: −23.9%**

Это плохой день, но не конец. Такое совпадение за всю историю бэктеста не случалось ни разу.

## Карта событий → стратегия

Бот должен определять тип события и направлять его в нужную стратегию:

```
СОБЫТИЕ: "Binance/Bybit Launchpool/HODLer/Megadrop анонс"
  └── Float < 25% && FDV > $300M && нет вестинга
      → СТРАТЕГИЯ 2: Launchpool Harvest
      → Открыть шорт за 12–24ч до листинга, плечо 3x

СОБЫТИЕ: "Новый листинг на Bybit/OKX/Binance (спот + фьючерс)"
  └── ≥3 фильтра из 5 (Float, FDV/MCap, Листинговая переоценка, Volume, Unlock)
      → СТРАТЕГИЯ 1: Token Splash Short
      → Ждать RSI>85 + Volume>300% + VWAP+40% + разворот, плечо 4x

СОБЫТИЕ: "TGE / Airdrop раздача"
  └── >50K получателей && 0 вестинг && Airdrop >8% supply && FDV >$500M
      → СТРАТЕГИЯ 4: TGE Short
      → Войти за 2ч до раздачи или на первом пике, плечо 2x

СОБЫТИЕ: "Новый листинг / уже торгующийся токен с FDV Ratio >8x"
  └── FDV/MCap > 8x && Float < 20% && нет real product
      → СТРАТЕГИЯ 3: VC-гравитация
      → Открыть шорт сразу после листинга, плечо 2x, горизонт 4–8 нед.

СОБЫТИЕ: "Одно событие попадает в несколько стратегий"
  └── Пример: Launchpool токен + FDV Ratio >8x
      → Launchpool Harvest приоритет (более точный тайминг)
      → VC-гравитация параллельно, если остаток капитала позволяет
```

## Архитектура единого бота на Go

Один Go-сервис, четыре модуля, один event loop.

```
cmd/
  bot/
    main.go              — точка входа, запуск всех модулей

internal/
  scanner/
    announcements.go     — парсинг анонсов Binance/Bybit/OKX
    classifier.go        — определение типа события
    tokenomics.go        — расчёт FDV ratio, float%, airdrop данных

  strategies/
    token_splash.go      — WebSocket + RSI/Volume/VWAP/BB логика
    launchpool.go        — плановое открытие шорта по расписанию
    vc_gravity.go        — трекинг unlock schedule, trailing stop
    tge_short.go         — мониторинг TGE событий

  risk/
    manager.go           — контроль открытых позиций, лимиты маржи
    position.go          — TP/SL/тайм-стоп управление

  exchange/
    bybit.go             — REST + WebSocket клиент Bybit
    okx.go               — REST клиент OKX
    binance.go           — REST клиент Binance

  logger/
    trades.go            — логирование сделок в CSV/SQLite
```

### Порядок разработки (что строить первым)

```
Неделя 1–2: scanner/announcements.go + scanner/classifier.go
  → Парсинг анонсов Binance/Bybit
  → Детект Launchpool, TGE, новых листингов
  → Алерт в Telegram при появлении события
  → БЕЗ автоматической торговли (только уведомления)

Неделя 3–4: strategies/launchpool.go + exchange/bybit.go
  → Открытие шорта по расписанию через Pre-Market API
  → Установка SL/TP/тайм-стоп
  → ПЕРВАЯ АВТОМАТИЧЕСКАЯ СДЕЛКА

Неделя 5–6: strategies/token_splash.go (WebSocket)
  → Подключение к kline.1.{SYMBOL}USDT
  → Расчёт RSI(7), Volume Ratio, VWAP, BB(20,2.5)
  → Сигнал → рыночный ордер

Неделя 7–8: risk/manager.go
  → Контроль: не более X% капитала в позициях
  → Запрет открытия новой позиции при превышении лимита
  → Тайм-стопы для всех стратегий

Месяц 3+: strategies/vc_gravity.go + strategies/tge_short.go
  → VC-gravity: daily job, трекинг unlock, trailing stop
  → TGE: классификатор событий из scanner
```

## Практический план на первые 3 месяца

### Месяц 1: Запуск Launchpool Harvest

**Цель:** $200 → $380–450

5. К концу месяца: 3–4 Launchpool трейда, из них 2–3 автоматических

**Что НЕ делаешь в первый месяц:**
- Не открываешь VC-гравитацию (нужна вся маржа для Launchpool/Splash)
- Не торгуешь на Binance (Bybit Pre-Market достаточно)
- Не торгуешь мемы, стейблы, SENT-подобные аномалии

### Месяц 2: Добавляем Token Splash

**Цель:** $400–450 → $800–1,000

   - WebSocket подключение к Bybit

4. К концу месяца: Token Splash дает 2–3 сделки в неделю

**Ключевой приоритет месяца 2:** не спешить с полным автоматом на Token Splash. Высокое плечо (4x) требует, чтобы ты видел как минимум 10–15 реальных сигналов, прежде чем доверять боту полностью.

### Месяц 3: Полная система

**Цель:** $900–1,000 → $1,500–2,000

1. Включаешь VC-гравитацию:
   - Находишь токен с FDV Ratio > 8x (новый листинг)
   - Открываешь шорт на 10–15% капитала, плечо 2x
   - Забываешь на 4–8 недель
2. TGE Short — ситуативно, когда событие квалифицируется
3. Все 4 стратегии работают параллельно

## Управление рисками системы

### Правило 1: Максимальная открытая маржа
```
Фаза 1 (до $500): 25% капитала → одна позиция
Фаза 2 (до $1,000): 45% капитала → две позиции
Фаза 3 ($1,000+): 60% капитала → три-четыре позиции
```

### Правило 2: Стоп-лосс — закон, не рекомендация
Ни одна стратегия, ни один актив не отменяет стоп. TUNA вырос на +2637% в декабре 2024. Без стопа один такой трейд уничтожал весь депозит. Бот исполняет стоп без эмоций — это его главная функция.

### Правило 3: Тайм-стоп — обязателен для всех стратегий
```
Token Splash:   24 часа
Launchpool:     48 часов
TGE Short:      7 дней
VC-гравитация:  60 дней
```
Если тезис не отработал за отведённое время — рынок говорит, что ты ошибся. Выходишь.

### Правило 4: Фандинг рейт — следить при долгих позициях
При VC-гравитации и TGE Short (дни/недели): если funding rate > 0.1% каждые 8 часов — позиция дорожает в удержании. Учитываешь в P&L.

### Правило 5: Бычий рынок = снижаешь плечо
В сильный бычий тренд (BTC > ATH, большинство альтов растёт) win rate Token Splash падает на 15–20 п.п. из-за ложных сигналов. Действие: снижаешь плечо с 4x до 2x или пропускаешь слабые сигналы.

## Итоговый финансовый прогноз системы

| Месяц | Стратегии | Реалистичный результат |
|---|---|---|
| 1 | Launchpool + ручной Token Splash | **$200 → $320–430** |
| 2 | Оба автоматизированы + TGE ситуативно | **$400 → $700–1,000** |
| 3 | Все 4 стратегии | **$800 → $1,400–2,000** |
| 6 | Полная система, оптимизированная | **$1,500 → $5,000–10,000*** |

*Оптимистичный диапазон требует: win rate Token Splash ≥70%, 6–8 сделок/месяц, строгое исполнение стопов, отсутствие крупных аномальных листингов (TUNA-сценарии). Реалистичный ориентир ближе к нижней границе каждого диапазона.

> **Почему цифры скорректированы:** VC-гравитация при корректном лимите 20% маржи (не 100%) даёт $200 → $290–360 за 3 месяца, а не $622. Token Splash остаётся основным двигателем роста системы.

Источником роста является **реинвест**: каждая следующая сделка использует бо́льший капитал. При win rate 75% и средней прибыли +80% маржи за сделку + 6 сделок в месяц — компаундинг делает всё остальное.

## Ответ на главный вопрос: с чего начать прямо сейчас

Завтра:
1. Открыть аккаунт на Bybit, создать API-ключ с правами Order/Position
2. Зайти на announcements.bybit.com — посмотреть ближайшие листинги вручную
3. Написать первую Go-программу: парсинг announcements.bybit.com раз в 5 минут → вывод в консоль

На этой неделе:
4. Дождаться Launchpool анонса → войти первый раз вручную по стратегии 2
5. Логировать: токен, дата, float%, FDV, цена входа, цена выхода, P&L

Всё остальное — следующие шаги, которые появятся сами, когда будет опыт первых сделок.

> **Система работает только если бот работает. Бот работает только если ты его написал. Напиши первую функцию сегодня.**
