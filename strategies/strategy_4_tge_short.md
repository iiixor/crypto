# Стратегия 4: TGE Short — Дамп на Token Generation Event

> **Стартовый капитал:** $200 | **Плечо:** 2x | **Горизонт сделки:** 1–7 дней
> **Биржи:** Bybit, OKX, Binance | **Тип:** шорт в момент максимального давления продавцов

---

## Философия

TGE (Token Generation Event) — момент, когда airdrop-токены становятся доступны. Это самый концентрированный момент продажи в истории любого токена.

**Верифицированная статистика (CoinLaw, 2025, выборка 2024 года):**
- **64%** получателей airdrop продают токены немедленно на TGE
- **88%** airdrop-токенов теряют стоимость в первые 3 месяца
- **66%** проектов показывают падение цены сразу после раздачи

Ты шортируешь в момент **максимального предложения при нулевой себестоимости у продавцов**.

## Чем отличается от Launchpool Harvest

| | Launchpool Harvest | TGE Short |
|---|---|---|
| Откуда давление | Фармеры CEX (ограниченный пул) | Сотни тысяч on-chain кошельков |
| Масштаб продавцов | 10,000–100,000 аккаунтов | 100,000–1,000,000+ кошельков |
| Предсказуемость | Очень высокая | Очень высокая |
| Скорость дампа | 24–48 часов | Часы–дни |
| Тип продавца | Стейкеры CEX | On-chain фармеры |

## Фильтры отбора

Все 5 фильтров — обязательные:

### Фильтр 1 — Масштаб раздачи
```
Количество получателей > 50,000 кошельков
```
Больше получателей = больше мелких продавцов без стратегии = быстрый дамп.

### Фильтр 2 — Отсутствие вестинга
```
Airdrop-доля разблокирована 100% на TGE (нет вестинга)
```
Вестинг даже на 3 месяца резко снижает давление продаж.

### Фильтр 3 — Доля airdrop
```
Airdrop allocation > 8% от total supply
```
Маленькая доля = маленькое давление. Нужно значимое количество токенов.

### Фильтр 4 — FDV
```
FDV на TGE > $500M
```
При маленьком FDV нет ликвидности для фьючерсов — невозможно войти в шорт.

### Фильтр 5 — Профиль получателей
```
Аirdrop farming-проект:
- Высокое число кошельков с минимальной активностью
- Много одноразовых адресов без истории DeFi-активности
```
Настоящие пользователи продают медленнее, чем фармеры.

## Логика входа

```
1. Объявление TGE → проверяешь 5 фильтров
2. За 2–4 часа до начала торгов:
   a. Открываешь шорт через фьючерс на Bybit/OKX
   b. Если Pre-Market цена доступна — входишь по Pre-Market

Или:
3. В первые 30 минут торгов:
   a. Ждёшь первого пика (RSI > 75 на 5-минутном таймфрейме)
   b. Входишь на первой медвежьей свече после пика
```

## Параметры ордера

| Параметр | Значение |
|---|---|
| Плечо | **2x** |
| Размер маржи | **20% капитала** |
| Стоп-лосс | **+12% от входа** |
| TP1 (50% позиции) | **−30% от входа** |
| TP2 (50% позиции) | **−50% от входа** |
| Тайм-стоп | **7 дней** |

**Мультипликаторы прибыли на маржу при 2x:**
- TP1 (−30% × 2x × 50%) = **+30%** маржи
- TP2 (−50% × 2x × 50%) = **+50%** маржи
- **Full Win** = **+80%** маржи
- **Partial Win** (только TP1) = **+30%** маржи
- **Stop Loss** (+12% × 2x) = **−24%** маржи

### Проверка расчёта:
```
Маржа $40, Нотионал $80 (2x)
TP1 на −30%: 30% × $80 × 50% = $12
TP2 на −50%: 50% × $80 × 50% = $20
Full Win: +$32 = +80% от маржи $40 ✓

Stop Loss на +12%: 12% × $80 = $9.60 = −24% от $40 ✓
```

## Что строить в Go

```go
// Мониторинг TGE событий

type AirdropEvent struct {
    Token         string
    TGEDate       time.Time
    Recipients    int64
    AirdropPct    float64  // % от total supply
    Vesting       int      // в месяцах (0 = сразу)
    FDV           float64
    IsQualified   bool
}

func qualifiesForShort(e AirdropEvent) bool {
    return e.Recipients > 50000 &&
           e.Vesting == 0 &&
           e.AirdropPct > 0.08 &&
           e.FDV > 500_000_000
}

// Источники данных для мониторинга:
// 1. Airdrops.io API / RSS
// 2. CoinMarketCap Events API
// 3. DeBank / Nansen on-chain данные
// 4. Twitter/X — поиск по "[TOKEN] airdrop claim"

// Алгоритм:
// 1. Детектируем TGE событие
// 2. Проверяем 5 фильтров
// 3. За 2 часа до TGE — входим в шорт
// 4. Устанавливаем TP1 (−30%), TP2 (−50%), SL (+12%)
// 5. Тайм-стоп: 7 дней
```

## Антипаттерны

```
НЕ входить, если:
[ ] Получатели — реальные активные пользователи (не фармеры)
    → продают медленно, ждут роста
[ ] Есть вестинг даже на 1 месяц
    → давление размывается
[ ] Проект анонсирует buyback или lock-up программу
    → VC и команда не продают, давление меньше
[ ] FDV < $500M
    → нет ликвидности для шорта через фьючерс (порог совпадает с Фильтром 4)
[ ] Нарратив токена совпадает с текущей хайп-темой рынка (AI, RWA, BTCFi)
    → покупатели могут превысить давление продавцов
[ ] Нет фьючерсного контракта с OI > $500K
    → невозможно нормально войти и выйти без проскальзывания
```

**Оптимальная комбинация:**

TGE Short — самостоятельно даёт невысокую доходность, но при совмещении с другими стратегиями добавляет дополнительные 30–80% прибыли без существенного увеличения нагрузки. Одна сделка в 2–4 недели, понятный тезис, чёткие параметры.

Главное отличие TGE Short от Launchpool: масштаб давления продавцов на порядок выше, дампы быстрее и глубже (NIGHT −89% за 7 дней), но события встречаются реже и требуют детального анализа on-chain данных.
