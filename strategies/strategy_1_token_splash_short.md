## Философия

Большинство новых токенов структурно переоценены в момент листинга. Маленький флоат, большой хайп, участники лонгпадов с нулевой или низкой себестоимостью — всё это создаёт предсказуемый памп, за которым следует такой же предсказуемый дамп.

**Реальная статистика (верифицировано):**
- 84.7% токенов, залистившихся на Bybit в 2025 году, упали ниже цены листинга
- Медианный ROI нового листинга: **0.23x** (потеря 77% стоимости)
- Январь 2026: из 26 новых листингов Bybit только 2 показали устойчивый рост

Ты не торгуешь против рынка. Ты торгуешь **по течению структурных сил**. Покупатели на листинге платят за эмоцию. Ты зарабатываешь на её угасании.

## Фильтры отбора токена

Токен квалифицируется для шорта при выполнении **3 из 5 условий** (данные проверяются на CoinMarketCap):

### Фильтр 1 — Float Ratio
```
Circulating Supply / Max Supply < 0.20
```
Меньше 20% токенов в обороте = огромный будущий навес продаж.

### Фильтр 2 — FDV / Market Cap
```
FDV / Market Cap > 7x
```
Рынок оценивает 10% саплая так, будто весь проект стоит в 7 раз дороже реального.

### Фильтр 3 — Листинговая переоценка
```
Цена открытия / Цена лончпада > 8x
```
Участники лончпада сидят с иксами и ждут ликвидности для выхода.

### Фильтр 4 — Volume Spike
```
24h Volume / Market Cap > 0.80 в первые 2 часа
```
Аномальный оборот = кульминация покупок, пик FOMO.

### Фильтр 5 — Unlock Schedule
```
Ближайший разлок команды/инвесторов < 90 дней
```
Будущее давление продаж подтверждено календарём. Источник: tokenunlocks.com

## Индикаторы входа (все 4 обязательны)

### RSI(7)
- Период: **7** (стандартный 14 запаздывает на новых листингах)
- Таймфрейм: **1m или 3m**
- Сигнал: RSI > **85**

### Volume Spike
- Объём текущей свечи > **300%** от среднего за последние 10 свечей
- Подтверждает, что кульминация покупок состоялась

### VWAP
- Якорь: момент открытия листинга
- Сигнал: цена выше VWAP на **40%+**
- Цель: первый тейк на возврате к VWAP

### Bollinger Bands (20, 2.5)
- Период: **20**, отклонение: **2.5** (2.0 даёт ложные сигналы при аномальной волатильности)
- Сигнал: цена вышла за верхнюю полосу и начала возврат внутрь коридора

### Условие входа в коде (Go):
```go
// Indicators — все четыре обязательных индикатора
type Indicators struct {
    RSI7           float64
    VolumeRatio    float64  // текущий объём / средний за 10 свечей
    VWAPDelta      float64  // (цена - VWAP) / VWAP
    BBUpper        float64  // верхняя полоса BB(20, 2.5)
    IsReversal     bool     // текущее закрытие < предыдущего
    PrevAboveBB    bool     // предыдущее закрытие > BBUpper (цена была выше BB)
}

func shouldShort(ind Indicators) bool {
    return ind.RSI7 > 85 &&
           ind.VolumeRatio > 3.0 &&
           ind.VWAPDelta > 0.40 &&
           ind.PrevAboveBB &&   // предыдущая свеча была выше верхней полосы BB
           ind.IsReversal        // текущая свеча закрылась ниже предыдущей (разворот)
}
```

> ⚠️ **Исправление:** Bollinger Bands описан как 4-й обязательный индикатор, но ранее отсутствовал в функции проверки. Добавлено поле `PrevAboveBB` — true если предыдущая свеча закрылась выше BBUpper. Это гарантирует вход только тогда, когда цена реально вышла за полосу и начала разворот, а не просто снижается.

**Критическое правило:** не входишь на пике, входишь на первой медвежьей свече после пробоя BB. Теряешь 2–3% от идеального входа. Экономишь на ложных сигналах и ликвидациях.

- Stop Loss: **−60%** маржи (4x × 15%)

## Архитектура бота на Go

### WebSocket endpoint Bybit
```go
// Мониторинг нового листинга
wss://stream.bybit.com/v5/public/linear
topic = "kline.1.{SYMBOL}USDT"  // 1-минутные свечи

// Структура данных
type Candle struct {
    Close  float64
    Volume float64
    Time   int64
}

type Indicators struct {
    RSI7        float64
    VolumeRatio float64  // текущий объём / средний за 10 свечей
    VWAPDelta   float64  // (цена - VWAP) / VWAP
    BBUpper     float64
    IsReversal  bool     // текущее закрытие < предыдущего
}
```

### Ключевые REST endpoints
```
POST /v5/order/create  — открытие позиции (market order)
POST /v5/order/create  — установка SL/TP (conditional order)
GET  /v5/position/list — мониторинг открытых позиций
```

### Логика исполнения
```
1. Мониторинг анонсов листингов (RSS/HTML парсинг каждые 5 минут)
2. Детект нового тикера → подключение к WebSocket
3. Накопление свечей → расчёт RSI(7), Volume Ratio, VWAP, BB(20,2.5)
4. Проверка всех 4 условий одновременно
5. Открытие шорта → рыночным ордером (market order)
6. Установка SL (+15%), TP1 (−25%, 50% позиции), TP2 (−50%, 50% позиции)
7. Тайм-стоп через 24 часа
8. Логирование: тикер, цена входа, выхода, P&L, причина закрытия
```

## Реальные риски

**Рост позиции = рост трения.** К третьему месяцу нотионал $680+. На мелком токене ты уже двигаешь стакан. Слипейдж вырастает с $5 до $20–30 за трейд.

**Фандинг рейт.** Когда все шортят, фандинг уходит в минус для шортов. При $680 нотионале: −$2 за 24 часа. Мелочь на одну сделку, но −$20 за месяц на масштабе.

**Аномальные листинги.** TUNA +2637%, SENT +рост в январе 2026. Один такой памп без стопа уничтожает 3–4 успешных трейда. **Стоп-лосс не обсуждается.**

**Бычий рынок снижает win rate.** В сильный бычий тренд RSI(7) > 85 даёт больше ложных сигналов. Win rate упадёт на 15–20 п.п.
